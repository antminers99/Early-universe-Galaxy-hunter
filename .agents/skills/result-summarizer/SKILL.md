# Result Summarizer

## Purpose

Summarize each phase of work in simple, clear language so anyone can understand what we did and what we found.

## When to Use

- After completing a phase of analysis
- When the user asks for a status update
- Before starting a new phase
- When preparing to share results

## Summary Structure

```
## ملخص المرحلة: [اسم المرحلة]

### ماذا عملنا
[وصف بسيط لما تم — بدون مصطلحات تقنية معقدة]

### ماذا وجدنا
[النتائج الرئيسية بنقاط واضحة]
- النتيجة 1
- النتيجة 2
- النتيجة 3

### ماذا يعني هذا
[تفسير بسيط — لماذا هذا مهم أو غير مهم]

### ما لم ينجح
[الأشياء التي جربناها ولم تعطِ نتائج]

### الخطوة التالية
[ماذا نقترح نعمل بعدها]

### مستوى الثقة
[كم نحن واثقون من النتائج — ربط مع Claim Limiter]
```

## Rules

- Use Arabic (Levantine dialect for tone, MSA for technical terms)
- Keep sentences short
- Avoid jargon — if technical term is needed, explain it
- Be honest about uncertainty
- Include both successes and failures
- Link to detailed reports for those who want more depth
